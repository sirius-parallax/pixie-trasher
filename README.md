## 📋 Краткая инструкция по установке

```bash
# 1. Установите зависимости
sudo apt update
sudo apt install reaver aircrack-ng wireless-tools python3-pip -y
pip3 install luma.oled

# 2. Установите модифицированный Reaver (обязательно!)
git clone https://github.com/t6x/reaver-wps-fork-t6x
cd reaver-wps-fork-t6x/src
./configure && make && sudo make install

# 3. Создайте скрипты (скопируйте содержимое выше)
nano /root/oled_display.py
nano /root/pixie_attacker.sh

# 4. Сделайте исполняемыми
chmod +x /root/oled_display.py /root/pixie_attacker.sh

# 5. Запустите
sudo /root/pixie_attacker.sh
## 📖 Краткое описание работы скрипта

### 🎯 Общая схема

```
Запуск → Проверка зависимостей → Поиск сетей → Выбор цели → Pixie Dust атака → Результат
```

---

### 📋 Пошаговая логика

#### 1. **Проверка зависимостей**
- Проверяет наличие `reaver` (модифицированная версия с поддержкой `-K`)
- Проверяет `airmon-ng` и `wash`
- Создаёт папку для результатов (`pixie_results`)

#### 2. **Запуск OLED (опционально)**
- Запускает `oled_display.py` в фоне
- Создаёт FIFO канал для отправки сообщений
- Если OLED не найден — работает без него

#### 3. **Поиск WPS сетей**
- Включает режим монитора на `wlan0`
- Запускает `wash --scan` на 25 секунд
- Парсит вывод, ищет сети с **Locked=No**
- Сохраняет в `BSSID|канал|сигнал|ESSID`
- Выключает режим монитора

#### 4. **Выбор цели**
- Показывает таблицу найденных сетей
- Ждёт выбора: `номер сети` / `A` (все) / `G` (хороший сигнал) / `0` (выход)
- **Автовыбор G через 15 секунд** (атакует только сети с сигналом ≥ -65)

#### 5. **Pixie Dust атака**
- Включает режим монитора
- Запускает `reaver -K` (Pixie Dust)
- Собирает данные с роутера (PKE, PKR, E-Hash, AuthKey)
- `pixiewps` вычисляет PIN оффлайн
- Получает WPA пароль
- Выключает режим монитора

#### 6. **Сохранение результатов**
- При успехе: PIN и пароль выводятся в терминал и на OLED
- Сохраняет в `pixie_results/cracked_passwords.txt`
- Создаёт отдельный файл для каждой взломанной сети

#### 7. **Пропуск уже взломанных**
- При повторном запуске проверяет `cracked_passwords.txt`
- Если BSSID уже есть — пропускает сеть (показывает `SKIPPED`)

---

### 🖥️ Вывод на OLED (4 строки)

| Строка | Что показывает | Пример |
|--------|----------------|--------|
| 1 | Счётчик + статус | `[2/5] Cracking...` |
| 2 | Название сети + сигнал | `Fiber.ru-147 -65` |
| 3 | Процесс / PIN | `Pixie Dust...` |
| 4 | Пароль | `PASS: MySecret` |

---

### 📊 Статусы атаки

| Статус | Что означает |
|--------|--------------|
| `CRACKED!` | Успех! PIN и пароль найдены |
| `FAILED` | Роутер не уязвим к Pixie Dust |
| `SKIPPED` | Сеть уже взломана ранее |
| `Not vulnerable` | Pixie Dust не сработал |
